axios

axios这个方法十分的简单，都是如如下形式

axios({
  url:'',
  method:'get',
  params:{

  }
}).then(res=>{
  console.log(res)
})

axios默认是get请求，params是get请求附带的参数，如果是post请求的话参数就放在data里面

axios.all

这个是如果要两个或者以上的请求都完成后再进行下一步

axios.all([axios({
  url: ''
}),axios({
  url:''
})]).then(res=>{
  console.log(res)
})

axios.spread

spread是对结果的延展

这里写一个案例

axios.all([axios({
  url:'http://101.200.180.194:8000/goods/'
}),axios({
  url:'http://101.200.180.194:8000/category/'
})]).then(res=>{
  console.log(res)
})

如果是如上的写法的话，得到的结果是

spread的功能就是

axios.all([axios({
  url:'http://101.200.180.194:8000/goods/'
}),axios({
  url:'http://101.200.180.194:8000/category/'
})]).then(axios.spread((res1,res2)=>{
  console.log(res1)
  console.log(res2)
}))

axios全局配置

比如axios在配置的时候有很多是重复的，比如超时时间、请求头、地址的重复字段如http:127.0.0.1等

如果都重复写很臃肿，所以可以用default属性

axios.defaults.baseURL='http://101.200.180.194:8000/'
axios.defaults.timeout=5000
axios.all([axios({
  url:'goods'
}),axios({
  url:'category'
})]).then(axios.spread((res1,res2)=>{
  console.log(res1)
  console.log(res2)
}))

这样还是可以请求到结果的

axios的实例

这个还是上面的那个全局配置问题，如果项目里面有两个或者以上的不同配置，比如请求头的地址不一样，那么如果不用全局配置写法的话，那么就会很多重复，如果用了全局配置的话，那么剩下的不同配置的文件就会报错

const instance1 = axios.create({
  baseURL:'http://101.200.180.194:8000/',
  timeout:5000
})

instance1({
  url:'goods'
}).then(res=>{
  console.log(res)
})

instance1({
  url:'category',
}).then(res=>{
  console.log(res)
})

const instance2 = axios.create({
  baseURL:'http://127.0.0.1/',
  timeout:5000
})

instance2({
  url:'person'
}).then(res=>{
  console.log(res)
})

这样就可以有两个不同的配置了

  console.log(err)
})

因为在axios里面的.then、.catch都是多余的，instance的返回值本身就是promise
